# 変圧器の温度予測および異常検知モデルの検証レポート (ETTh)

本プロジェクトでは、変圧器の稼働データ（ETTh）を用い、温度予測および異常発熱のアラート機能の実用化に向けた検証を行いました。以下にその検証プロセスと結論をまとめます。

1. 特徴量の選定
事前分析（EDA）の結果、目的変数（油温）と相関が高かった「MULL（中圧無効電力）」の特徴量を主軸として、LightGBMによる回帰モデルを構築しました。

2. 温度予測モデルの基本性能
純粋な温度予測の精度としては、1時間後の予測であれば実際の波形にそこそこ綺麗に追従できています。ただし、予測期間（Horizon）を3時間、6時間、12時間と長くしていくにつれて、当然ながら予測値の精度は粗くなっていく結果となりました。

3. 現場運用を見据えた「しきい値」判定の壁
現場での実運用を考慮し、単に温度を予測するだけでなく「危険なしきい値を超えたらアラートを出す」機能を追加して検証しました。
・絶対的しきい値（例：固定の温度）：見かけ上の精度は出ますが、変圧器の温度は季節（気候変動）やサイズに大きく影響されるため、固定値のしきい値は現場運用においてあまり意味がありません。
・相対的しきい値（動的変動）：そこで季節変動を加味した相対的なしきい値を設定した結果、アラートの精度（特に突発的な異常の見逃し）が著しく低下し、実際の現場で運用できるレベルの精度ではないことが判明しました。

4. 突発的な変化への脆弱性と仮説
モデルの予測結果を分析した結果、全体のトレンドを追うことはできても「突発的な温度変化（スパイク）」に対して特に弱いことがわかりました。このことから、異常発熱の根本原因は内部の電力負荷（MULL等）の蓄積ではなく、外部からの要因にあるのではないかという仮説に至りました。

5. 結論と今後の改善案
突発的な外部要因を捉えるため、外気温や天候などの気象APIと連携してデータを追加できれば、精度を実用レベルまで上げられる可能性は高いと考えています。
しかし、現状利用しているデータセット（ETTh）は変圧器の所在地が不明であるため、該当地域の外気温データを正確に取得・結合することが不可能です。
したがって、現在の内部データセットのみを用いたアプローチでは、これ以上の精度向上および現場での実用化は困難であると結論づけました。
